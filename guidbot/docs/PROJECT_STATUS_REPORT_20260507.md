### 3-2. 즉시 대응 필요 항목

#### ❶ LLM SQL UNION 차단 미적용 (CRITICAL)
```
파일: llm/sql_generator.py:96
현재: # (re.compile(r"\bUNION\b", ...), "UNION 차단"),  ← 주석 처리됨
위험: SELECT 1, USER, password FROM DUAL 형태 injection 가능
조치: UNION 패턴 재활성화 + oracle_whitelist_tables 강제 설정
```

#### ❷ HTTPS 미구성 (HIGH)
```
현재: HTTP 평문 운영
위험: Oracle 비밀번호, Gemini API 키 네트워크 노출
조치: Nginx 리버스 프록시 + 병원 내부 CA 인증서
```

## 9. 결론

### 완료된 것 (오늘 기준)

이번 작업에서 **Critical 2개·High 1개·Medium 3개** 항목이 해결되었습니다.

- 가장 위험했던 **관리자 기본 비밀번호 하드코딩**이 제거되어 `ADMIN_PASSWORD` 환경변수 미설정 시 앱 기동이 차단됩니다.
- **30분 세션 만료** 도입으로 공용 PC 환경에서의 세션 탈취 위험이 감소했습니다.
- **Oracle 쿼리 병렬화**로 실시간 탭 렌더링 시간이 2–5초에서 0.5–1초로 개선되었습니다.
- **31개 단위 테스트**가 100% 통과하며 핵심 보안 로직의 회귀 감지 기반이 마련되었습니다.

### 가장 시급한 잔존 위험

1. **LLM SQL UNION 미차단** — 의도적으로 주석 처리된 상태. 공격자가 챗봇을 통해 `UNION SELECT USER, password FROM DBA_USERS` 형태의 쿼리를 실행시킬 수 있습니다. **즉시 재활성화 필요.**

2. **자동 재시작 없음** — OOM·예외 종료 시 수동 개입 전까지 서비스 중단. `systemd Restart=always` 등록만으로 해결 가능합니다.

3. **HTTPS 미구성** — 내부망이라도 Nginx + 내부 CA 인증서 구성 권장. Oracle 비밀번호가 HTTP로 노출됩니다.

### 운영 투입 판정

> **⚠️ 조건부 운영 가능**  
> UNION 차단 재활성화 + systemd 등록 완료 후 **Pilot 운영 투입 가능** 수준.  
> HTTPS·Docker·CI/CD는 본격 운영 전 완료 권장.
