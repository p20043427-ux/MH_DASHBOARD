## 1. 시스템 요구사항

### 최소 사양

| 항목 | 최소 | 권장 |
|------|------|------|
| OS | Windows 10 64-bit | Windows 11 Pro |
| CPU | Intel i5 6세대 이상 | Intel i5-10500 이상 |
| RAM | 8 GB | **16 GB 이상** |

제공받은 설치 패키지(ZIP)를 설치 폴더에 압축 해제합니다.

1. [https://aistudio.google.com](https://aistudio.google.com) 접속 (Google 계정 필요)

   형식 예시: `AIzaSyXXXXXXXXXXXXXXXXXXXXXXXX`

> 안정적 운영을 위해 API 키를 **2~5개** 발급받는 것을 권장합니다.

## 5. 환경 설정 (.env)

### 5-1. .env 파일 생성

`.env.example`을 복사하여 `.env` 파일을 만듭니다.

```cmd
copy .env.example .env
```

### 5-2. .env 필수 항목 편집

메모장(notepad) 또는 VS Code로 `.env` 파일을 열어 수정합니다.

```ini
# =====================================================
# [필수] Google Gemini API 키
# =====================================================
GOOGLE_API_KEY="AIzaSy여기에_발급받은_키_입력"

# 키가 여러 개인 경우 순번대로 입력 (선택사항, 권장)
# GOOGLE_API_KEY_2="AIzaSy두번째키"
# GOOGLE_API_KEY_3="AIzaSy세번째키"

# =====================================================
# [필수] 관리자 비밀번호 (기본값 반드시 변경)
# =====================================================
ADMIN_PASSWORD="병원명_강력한비밀번호12자이상"

# =====================================================
# [필수] Oracle DB 연결
# =====================================================
ORACLE_ENABLED=true
ORACLE_HOST=192.168.x.x          # Oracle DB 서버 IP
ORACLE_PORT=1521
ORACLE_SERVICE_NAME=ORCL         # 서비스명 (DBA에게 확인)
ORACLE_USER=guidbot_readonly      # SELECT 전용 계정
ORACLE_PASSWORD="DB계정비밀번호"

# =====================================================
# [선택] Oracle 11g 이하 환경 (Thick Mode)
# =====================================================
# ORACLE_THICK_MODE=true
# ORACLE_CLIENT_LIB_DIR=C:\oracle\instantclient_11_2

# =====================================================
# [선택] 앱 제목 변경
# =====================================================
APP_TITLE="병원명 가이드봇"
```

### 5-3. config/settings.py 경로 수정

`config/settings.py` 파일을 열어 `_BASE_DIR` 경로를 실제 설치 경로로 수정합니다.

```python
# config/settings.py 상단 (약 10번째 줄)
_BASE_DIR = Path(r"D:\guidbot")   # ← 실제 설치 경로로 변경
```

## 6. Oracle DB 연동 설정

### 6-1. Oracle 계정 생성 (DBA 작업)

병원 Oracle DBA에게 아래 내용으로 계정 생성을 요청합니다.

CREATE USER guidbot_readonly IDENTIFIED BY "안전한비밀번호";

-- 2. 연결 권한 부여
GRANT CREATE SESSION TO guidbot_readonly;

-- 3. VIEW 접근 권한 부여 (아래 목록의 VIEW들에 대해)
GRANT SELECT ON 병원스키마.V_WARD_DEPT_STAY      TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_BED_DETAIL     TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_OP_STAT        TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_KPI_TREND      TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_YESTERDAY      TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_DX_TODAY       TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_DX_TREND       TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_ADMIT_CANDIDATES    TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_WARD_ROOM_DETAIL    TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_FINANCE_TODAY       TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_OVERDUE_STAT        TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_FINANCE_BY_INS      TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_OPD_KPI            TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_OPD_BY_DEPT        TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_OPD_HOURLY_STAT    TO guidbot_readonly;
GRANT SELECT ON 병원스키마.V_NOSHOW_STAT        TO guidbot_readonly;
-- (재무 대시보드 VIEW 추가분)
GRANT SELECT ON 병원스키마.V_OPD_DEPT_STATUS    TO guidbot_readonly;

GRANT SELECT ON 병원스키마.V_FINANCE_TREND      TO guidbot_readonly;

### 6-2. VIEW 스키마명 변경

병원별로 Oracle 스키마명이 다릅니다. 소스 코드에서 `JAIN_WM`을 실제 스키마명으로 교체합니다.

```cmd
# PowerShell로 일괄 교체
(venv) D:\guidbot> powershell -Command "(Get-Content ui\hospital_dashboard.py) -replace 'JAIN_WM', '병원스키마명' | Set-Content ui\hospital_dashboard.py"
(venv) D:\guidbot> powershell -Command "(Get-Content ui\finance_dashboard.py)  -replace 'JAIN_WM', '병원스키마명' | Set-Content ui\finance_dashboard.py"
```

또는 .env에서 스키마를 설정합니다:

```ini
ORACLE_SCHEMA=병원스키마명
```

### 6-3. Oracle 11g 이하 (Thick Mode) 설정

Oracle 12c 미만(10g, 11g) 환경에서는 Instant Client 설치가 필요합니다.

1. Oracle 공식 사이트에서 **Instant Client 11.2 (64-bit Basic Lite)** 다운로드
   - URL: https://www.oracle.com/database/technologies/instant-client/winx64-64-downloads.html
2. `C:\oracle\instantclient_11_2` 에 압축 해제
3. `.env` 수정:

```ini
ORACLE_THICK_MODE=true
ORACLE_CLIENT_LIB_DIR=C:\oracle\instantclient_11_2
```

### 6-4. Oracle 연결 테스트

```cmd
(venv) D:\guidbot> python -c "
import oracledb, os
from dotenv import load_dotenv
load_dotenv()
conn = oracledb.connect(
    user=os.getenv('ORACLE_USER'),
    password=os.getenv('ORACLE_PASSWORD'),
    dsn=f\"{os.getenv('ORACLE_HOST')}:{os.getenv('ORACLE_PORT')}/{os.getenv('ORACLE_SERVICE_NAME')}\"
)
print('Oracle 연결 성공:', conn.version)
conn.close()
"
```

## 10. 자동 시작 등록

서버 재부팅 후 자동으로 서비스가 시작되도록 Windows 작업 스케줄러에 등록합니다.

### 10-1. 작업 스케줄러 등록

PowerShell (관리자 권한):

```powershell
$action  = New-ScheduledTaskAction -Execute "D:\guidbot\start.bat" -WorkingDirectory "D:\guidbot"
$trigger = New-ScheduledTaskTrigger -AtStartup
$settings = New-ScheduledTaskSettingsSet -RunOnlyIfNetworkAvailable -StartWhenAvailable
Register-ScheduledTask -TaskName "GuidBot Startup" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest -Force
```

### 10-2. 매일 새벽 재시작 (권장)

메모리 누적 방지를 위해 매일 자동 재시작을 권장합니다:

```powershell
$trigger2 = New-ScheduledTaskTrigger -Daily -At "06:00AM"
Register-ScheduledTask -TaskName "GuidBot DailyRestart" -Action $action -Trigger $trigger2 -Settings $settings -RunLevel Highest -Force
```

## 11. 트러블슈팅

### 오류: `ModuleNotFoundError`

```cmd
pip install -r requirements.txt --force-reinstall
```

### 오류: `UnicodeDecodeError` (requirements.txt 설치 시)

```cmd
pip install -r requirements.txt -v 2>&1 | head
```

requirements.txt 파일 인코딩 문제입니다. UTF-8 BOM으로 저장되어 있는지 확인합니다.

### 오류: `ORA-12541: TNS:no listener`

- Oracle 서버 IP/포트/서비스명 확인
- Oracle DB 서버 방화벽에서 1521 포트 허용 여부 확인

### 오류: `ORA-01017: invalid username/password`

`.env`의 `ORACLE_USER`, `ORACLE_PASSWORD` 확인.

### 오류: `DPY-6001: cannot connect to database` (Thick Mode 필요)

`.env`에 추가:

```ini
ORACLE_THICK_MODE=true
ORACLE_CLIENT_LIB_DIR=C:\oracle\instantclient_11_2
```

### 포트 충돌: `Port 8501 is already in use`

```cmd
# 사용 중인 프로세스 확인
netstat -ano | findstr :8501

# PID 확인 후 종료
taskkill /PID [PID번호] /F
```

### AI 챗봇 응답 없음: `GOOGLE_API_KEY` 오류

1. `.env`의 API 키 값 확인 (앞뒤 공백 없어야 함)
2. API 키 유효성 확인: https://aistudio.google.com

### 메모리 부족 (`Killed` 또는 앱 자동 종료)

- RAM 16GB 이상 확보
- 챗봇(main.py)과 대시보드를 분리된 서버에서 실행
- `ORACLE_POOL_MAX=5` 로 낮춰 DB 연결 수 제한
