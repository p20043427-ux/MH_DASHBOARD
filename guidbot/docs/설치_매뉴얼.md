### 1-3. RAG 파이프라인 아키텍처

```
사용자 질문
    │
    ▼
[QueryRewriter]  ← 한국어 구어체 정제, 조사 제거
    │
    ▼
[캐시 조회]  ── HIT ──→  즉시 반환 (TTL 10분, 최대 100개)
    │ MISS
    ▼
[검색 모드 분기]
    │
    ├─ ⚡ fast     : FAISS top-5 (CE 없음) → 0.5~1.5초
    │
    ├─ ⚖️ standard : HybridRetriever (FAISS 70% + BM25 30%)
    │               top-8 → Cross-Encoder Rerank → top-3 → 2.5~4.0초
    │
    └─ 🧠 deep     : Multi-Query 확장 (3개 관점)
                    각 10개 Hybrid 검색 → 중복 제거 → CE Rerank → top-5
                    CoT 컨텍스트 구축 → 4.0~7.0초

    ↓ 검색된 청크 + 컨텍스트
[Gemini 2.5 Flash LLM]  ← 스트리밍 응답
    │
    ▼
사용자 화면 (실시간 스트리밍 출력)
```

**HybridRetriever 가중치 병합 (RRF)**

```
최종 점수 = FAISS 의미 검색 × 0.70
           + BM25 키워드 검색 × 0.30
           → Reciprocal Rank Fusion(RRF) 방식 병합
```

**임베딩 모델**: `jhgan/ko-sroberta-multitask` (HuggingFace, ~350MB, 로컬 실행)  
**Cross-Encoder**: `bongsoo/kpf-cross-encoder-v1` (리랭킹 정확도 향상)  
**LLM**: Google Gemini 2.5 Flash (API, thinking 비활성 — 속도 우선)

## 7. Oracle DB 연동 설정

### 7-1. Oracle 계정 생성 (DBA 작업)

병원 Oracle DBA에게 아래 SQL로 전용 계정 생성을 요청합니다.

```sql
-- 1. 전용 계정 생성 (SELECT 전용)
CREATE USER guidbot_readonly IDENTIFIED BY "안전한비밀번호";
GRANT CREATE SESSION TO guidbot_readonly;

-- 2. 병동 대시보드용 VIEW 권한 (dashboard_app.py)
GRANT SELECT ON 스키마.V_WARD_DEPT_STAY      TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_BED_DETAIL     TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_OP_STAT        TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_KPI_TREND      TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_YESTERDAY      TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_DX_TODAY       TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_DX_TREND       TO guidbot_readonly;
GRANT SELECT ON 스키마.V_ADMIT_CANDIDATES    TO guidbot_readonly;
GRANT SELECT ON 스키마.V_WARD_ROOM_DETAIL    TO guidbot_readonly;
```

## 8. AI 지식베이스 구축

AI 챗봇이 병원 규정집 PDF를 학습하여 질의응답에 활용하는 FAISS 벡터 DB를 구축합니다.

### 8-1. 규정집 PDF 준비

```

    ├── 의약품관리지침.pdf
    └── ... (병원 규정 PDF 전체)
```

### 8-2. 지식베이스 구축 실행

**기본 방식 (권장)**:
```cmd
python build_db.py --no-sync
```

**표(Table) 구조 보존 방식** (NEDIS 코드표 등 표가 많은 규정집):
```cmd
python build_db.py --no-sync --use-markdown
```

**주요 옵션 정리**:

| 옵션 | 설명 |
|------|------|
| `--no-sync` | Google Drive 동기화 건너뜀 (로컬 PDF만 사용) |
| `--use-markdown` | PDF → Markdown 변환 후 청킹 (표 구조 보존) |
| `--chunk-size 800` | 청크 최대 크기 (기본 800자) |
| `--overlap 150` | 청크 간 중복 크기 (기본 150자) |
| `--dept 간호부` | 특정 부서만 재구축 (부분 업데이트) |
| `--dept all` | 모든 부서 재구축 + 마스터 병합 |
| `--no-merge` | 재구축 후 마스터 병합 건너뜀 |

**소요 시간**:
- AI 임베딩 모델 최초 다운로드: 5~10분 (~350MB)
- PDF 처리 및 인덱싱: 규정집 100페이지 기준 약 3~10분

## 11. 자동 시작 등록

서버 재부팅 후 자동으로 서비스가 시작되도록 Windows 작업 스케줄러에 등록합니다.

### 11-1. 부팅 시 자동 시작

PowerShell (관리자 권한):

```powershell
$action   = New-ScheduledTaskAction -Execute "D:\guidbot\start.bat" -WorkingDirectory "D:\guidbot"
$trigger  = New-ScheduledTaskTrigger -AtStartup
$settings = New-ScheduledTaskSettingsSet -RunOnlyIfNetworkAvailable -StartWhenAvailable
Register-ScheduledTask -TaskName "GuidBot Startup" `
    -Action $action -Trigger $trigger -Settings $settings `
    -RunLevel Highest -Force
```

### 11-2. 매일 새벽 자동 재시작 (권장)

메모리 누적 방지를 위해 매일 새벽 재시작을 권장합니다.

```powershell
$trigger2 = New-ScheduledTaskTrigger -Daily -At "06:00AM"
Register-ScheduledTask -TaskName "GuidBot DailyRestart" `
    -Action $action -Trigger $trigger2 -Settings $settings `
    -RunLevel Highest -Force
```

## 12. 트러블슈팅

### `ModuleNotFoundError`
```cmd
pip install -r requirements.txt --force-reinstall
```

### `ORA-12541: TNS:no listener`
- Oracle 서버 IP / 포트 1521 / 서비스명 확인
- Oracle DB 서버 방화벽 1521 포트 허용 여부 확인

### `ORA-01017: invalid username/password`
- `.env`의 `ORACLE_USER`, `ORACLE_PASSWORD` 확인

### `DPY-6001: cannot connect to database` (Thick Mode 필요)
```ini
# .env 추가
ORACLE_THICK_MODE=true
ORACLE_CLIENT_LIB_DIR=C:\oracle\instantclient_11_2
```

### `Port 8501 is already in use`
```cmd
netstat -ano | findstr :8501
taskkill /PID [PID번호] /F
```

### AI 챗봇 응답 없음: `GOOGLE_API_KEY` 오류
1. `.env` 키 값 앞뒤 공백 없는지 확인
2. 키 유효성 확인: https://aistudio.google.com

### 임베딩 모델 다운로드 실패 (인터넷 차단 환경)

```cmd
# 인터넷 연결 PC에서 사전 다운로드 후 복사
# HuggingFace 캐시 경로: C:\Users\유저명\.cache\huggingface\
```

### 메모리 부족 (앱 자동 종료)
- RAM 16GB 이상 확보 필요

- AI 챗봇(8502)과 대시보드를 분리된 서버 운영 고려
