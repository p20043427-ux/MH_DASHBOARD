### 1-3. RAG 파이프라인 아키텍처

```
사용자 질문
    │
    ▼
[QueryRewriter]  ← 한국어 구어체 정제, 조사 제거
    │
    ▼
[캐시 조회]  ── HIT ──→  즉시 반환 (TTL 10분, 최대 100개)
    │ MISS
    ▼
[검색 모드 분기]
    │
    ├─ ⚡ fast     : FAISS top-5 (CE 없음) → 0.5~1.5초
    │
    ├─ ⚖️ standard : HybridRetriever (FAISS 70% + BM25 30%)
    │               top-8 → Cross-Encoder Rerank → top-3 → 2.5~4.0초
    │               ※ BM25 디스크 캐시: warm start ~0.3초 (vs cold 3~8초)
    │
    └─ 🧠 deep     : Multi-Query 확장 (3개 관점)
                    각 10개 Hybrid 검색 → 중복 제거 → CE Rerank → top-5
                    CoT 컨텍스트 구축 → 4.0~7.0초

    ↓ 검색된 청크 + 컨텍스트
[Gemini 2.5 Flash LLM]  ← 스트리밍 응답
    │
    ▼
사용자 화면 (실시간 스트리밍 출력)
```

**HybridRetriever 가중치 병합 (RRF)**

```
최종 점수 = FAISS 의미 검색 × 0.70
           + BM25 키워드 검색 × 0.30
           → Reciprocal Rank Fusion(RRF) 방식 병합

BM25 인덱스: 최초 구축 후 디스크 캐시 저장
  경로: vector_store/.bm25_cache_{hash8}.pkl
  캐시 HIT: ~0.3초 / MISS(최초): 3~8초
```

**임베딩 모델**: `jhgan/ko-sroberta-multitask` (HuggingFace, ~350MB, 로컬 실행)
**Cross-Encoder**: `bongsoo/kpf-cross-encoder-v1` (리랭킹 정확도 향상)
**LLM**: Google Gemini 2.5 Flash (API, thinking 비활성 — 속도 우선)

## 5. 프로그램 설치

### 5-1. pip 업그레이드

```cmd
python -m pip install --upgrade pip setuptools wheel
```

### 5-2. 패키지 일괄 설치

```cmd
pip install -r requirements.txt
```

CPU 환경(GPU 없음)에서 `torchvision` 설치 시 PyTorch CPU 인덱스를 사용합니다:

```cmd
pip install torchvision --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
```

> `torchvision`은 GPU 버전 torch와 함께 설치하면 CUDA 버전을 자동 선택합니다.
> CPU 환경(현 서버)은 반드시 `--index-url https://download.pytorch.org/whl/cpu` 옵션을 사용합니다.

| `--dept all` | 모든 부서 재구축 + 마스터 병합 |

**소요 시간**:
- AI 임베딩 모델 최초 다운로드: 5~10분 (~350MB)
- PDF 처리 및 인덱싱: 규정집 100페이지 기준 약 3~10분
